# 合同签署实现建议

## 1. 基础版，不接第三方电子签 API

适用场景：
- 先快速上线
- 接受“平台留痕签署”而非强合规 CA 电子签

实现方式：
- 手机端直接在合同页手写签字
- 服务端保存：
  - 合同最终 HTML
  - 合同编号
  - 签署人姓名
  - 签字图片
  - IP
  - User-Agent
  - 签署时间
  - 签署前 challenge token
  - 合同内容 SHA-256 哈希
- 合同签署完成后生成 PDF 快照并归档
- 签署页增加三项勾选：
  - 已阅读全部条款
  - 已确认并入本金收益金额
  - 已确认法律服务费与逾期规则

结论：
- 不需要接第三方 API 就能做
- 但法律证明力弱于合规电子签

## 2. 增强版，桌面端扫码到手机签

实现方式：
- 桌面端打开合同时，服务端创建 `sign_session`
- 生成一次性签署链接
- 桌面端展示二维码
- 手机扫码后打开专属签署页
- 签署成功后通过轮询或 WebSocket 回推桌面端状态

必须补的安全点：
- 签署链接短时有效
- 单次使用
- 必须绑定合同 ID、客户 ID、设备切换日志
- 扫码后桌面端只显示结果，不允许桌面直接代签

## 3. 指纹/面容

浏览器里所谓“获取指纹”通常分两种：

1. 设备指纹
- 指浏览器环境指纹，不是生物识别
- 可用于风控
- 不能当作客户按了手指签署

2. 生物识别确认
- 需要 WebAuthn 或原生 App 的 Face ID / Touch ID / Android BiometricPrompt
- Web 端可以做“签署前二次确认”
- 返回的是一段加密断言，不会拿到真实指纹图片

建议：
- 如果你要“按一次指纹”的真实上线方案，优先做 WebAuthn 二次确认
- 如果你要更强合规效力，再接第三方电子签平台

## 4. 什么时候必须接第三方电子签

建议接第三方的场景：
- 要求更强司法证据能力
- 需要实名校验、人脸核验、CA 证书、可信时间戳
- 需要 PDF 骑缝章、证书链和司法验签报告

常见能力：
- 实名认证
- 活体/人脸
- CA 证书签名
- 时间戳
- PDF 固化
- 存证报告

## 5. 推荐落地顺序

1. 先做平台内最终 HTML/PDF 固化、内容哈希、签署会话、扫码到手机签
2. 再做 WebAuthn 二次确认
3. 最后按业务合规要求接第三方电子签
